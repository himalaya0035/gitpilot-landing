Getting Started with GitPilot

Set up your visual Git environment in minutes. Choose the installation method that works best for your workflow.

Quick Start with Docker Recommended

The fastest and most reliable way to run GitPilot. It ensures you have all the necessary dependencies isolated from your host system.

  1. Ensure you have Docker and Docker Compose installed on your machine.
  2. Create a docker-compose.yml file in your desired workspace directory and add the following configuration: 
    version: '3.8'

services:
  gitpilot:
    image: himalaya0035/gitpilot:latest
    ports:
      - "3000:3000"
    volumes:
      - ./your-repo:/app/repo # Map your local repo here
    environment:
      - NODE_ENV=production
  3. Run the application in detached mode: 
    docker-compose up -d
  4. Open your browser and navigate to http://localhost:3000 to start visualizing your Git tree.

Note: Make sure to update the volume mapping ./your-repo:/app/repo to point to the actual path of the Git repository you want to analyze.

Manual Setup

If you prefer to run GitPilot natively or want to contribute to the development, you can clone the repository directly.

  1. Clone the GitHub repository: 
    git clone https://github.com/himalaya0035/GitPilot.git
cd GitPilot
  2. Install the necessary dependencies (requires Node.js v18+): 
    npm install
  3. Start the development server: 
    npm run dev
